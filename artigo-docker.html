Introdução ao Docker: containers para desenvolvedores

Por Pedro 25 de Março de 2025

Se você já ouviu a frase "na minha máquina funciona", o Docker é a solução. Containers revolucionaram a forma como desenvolvemos, testamos e implantamos aplicações. Neste guia, vamos entender o que é Docker, por que usá-lo e como começar.

O que é Docker?

Docker é uma plataforma de código aberto que permite empacotar aplicações e todas as suas dependências em unidades padronizadas chamadas containers. Um container é um ambiente isolado e leve que roda sobre o sistema operacional do host, sem precisar de uma máquina virtual completa.

"Containers são para aplicações o que contêineres de transporte são para mercadorias — uma forma padronizada de empacotar e transportar qualquer coisa, para qualquer lugar."

Docker vs Máquinas Virtuais

Diferente de máquinas virtuais (VMs), que simulam um sistema operacional inteiro, containers compartilham o kernel do host e isolam apenas a aplicação e suas dependências. Isso resulta em:

  • Início mais rápido — Containers iniciam em segundos, VMs em minutos
  • Menos recursos — Containers usam muito menos memória e CPU
  • Maior portabilidade — O mesmo container roda em qualquer lugar com Docker instalado
  • Mais densidade — É possível rodar dezenas de containers onde caberiam poucas VMs

Conceitos fundamentais

Imagem (Image)

Uma imagem Docker é um template somente leitura que contém tudo necessário para rodar uma aplicação: código, runtime, bibliotecas, variáveis de ambiente e arquivos de configuração. Pense nela como uma "receita" para criar containers.

Container

Um container é uma instância em execução de uma imagem. Você pode criar múltiplos containers a partir da mesma imagem, cada um isolado dos demais.

Dockerfile

É um arquivo de texto com instruções para construir uma imagem Docker. Cada instrução cria uma "camada" na imagem:

# Exemplo de Dockerfile para uma aplicação Node.js
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["node", "server.js"]

Docker Compose

Ferramenta para definir e gerenciar aplicações com múltiplos containers. Usando um arquivo docker-compose.yml, você define todos os serviços, redes e volumes da aplicação:

# docker-compose.yml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db

  db:
    image: postgres:16
    environment:
      POSTGRES_DB: meuapp
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: senha123
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:

Comandos essenciais

Os comandos mais usados no dia a dia com Docker:

# Baixar uma imagem do Docker Hub
docker pull node:20

# Listar imagens locais
docker images

# Criar e rodar um container
docker run -d -p 3000:3000 --name meu-app minha-imagem

# Listar containers em execução
docker ps

# Parar um container
docker stop meu-app

# Ver logs de um container
docker logs meu-app

# Construir imagem a partir de um Dockerfile
docker build -t minha-imagem .

# Docker Compose - subir todos os serviços
docker compose up -d

# Docker Compose - derrubar tudo
docker compose down

Por que usar Docker no desenvolvimento?

  1. Ambiente consistente — Todos os desenvolvedores da equipe trabalham no mesmo ambiente, eliminando o "funciona na minha máquina"
  2. Onboarding rápido — Novos membros da equipe rodam um docker compose up e já estão prontos
  3. Isolamento — Cada projeto pode ter suas próprias versões de Node, Python, banco de dados, etc.
  4. Testes reproduzíveis — Ambientes de teste idênticos ao de produção
  5. CI/CD simplificado — Pipelines de integração contínua ficam mais simples e confiáveis

Docker na prática: exemplo completo

Vamos criar uma aplicação web simples com Node.js e PostgreSQL usando Docker Compose. A estrutura do projeto seria:

meu-projeto/
├── docker-compose.yml
├── Dockerfile
├── package.json
├── server.js
└── .dockerignore

O arquivo .dockerignore funciona como o .gitignore, evitando que arquivos desnecessários sejam copiados para a imagem:

node_modules
npm-debug.log
.git
.env

Docker Hub e registro de imagens

O Docker Hub é o registro público padrão, com milhares de imagens oficiais prontas para uso: Node.js, Python, PostgreSQL, Redis, Nginx e muito mais. Você também pode publicar suas próprias imagens.

Conclusão

Docker é uma ferramenta essencial no arsenal de todo desenvolvedor moderno. Ele simplifica ambientes de desenvolvimento, torna deploys mais confiáveis e facilita a colaboração em equipe. Se você ainda não usa Docker, comece com um projeto pequeno e vá explorando — os benefícios são enormes.
